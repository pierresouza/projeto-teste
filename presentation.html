⚠️ O Problema

A Crise da Geração Direta de Código

Por que a maioria dos devs fracassa ao usar IAs para codificar?

O Anti-Padrão Comum
$ Usuário: "Crie um sistema de reservas de
hotel com banco de dados, login,
pagamento e dashboard admin" ────────────────── ⚠ IA alucina regras de negócio ✗ Código sem padrões definidos ✗ Arquitetura inventada ✗ Zero documentação gerada
⚡

A Ilusão da Velocidade

Gerar arquivos de código diretamente parece rápido, mas cria dívida técnica imediata e código sem âncora de decisões.

🎲

Adivinhação de Requisitos

Sem regras claras escritas, a IA inventa regras de negócio, limites de segurança e chamadas de API que não existem.

🧠

Falta de Âncora de Contexto

A IA perde a memória de todo o racional à medida que o chat cresce. Cada sessão reinicia o caos.

🧅 O Framework

Como funciona o Onion Mini?

O Onion Mini é uma arquitetura de regras e personas que transforma qualquer assistente de IA em um orquestrador disciplinado de desenvolvimento.

📋

O Master Prompt

O arquivo ONION-MASTER-PROMPT.md é o sistema operacional lógico da IA. Ele define todas as personas, regras de fluxo e comportamentos adaptativos.

⚡ Otimizado para Limitações Reais

Reduzido de 6 para apenas 3 arquivos principais, respeitando o limite de 5 arquivos do plano grátis do ChatGPT sem perder nenhuma funcionalidade.

🧑‍💼 @product

Ciclo de Produto — Especificando o Negócio

1️⃣

Fase Collect (Coleta)

A IA valida qual "Dor do Cliente" do business-context-lite.md a ideia resolve. Se não estiver claro, faz até 3 perguntas cirúrgicas de refinamento.

2️⃣

Fase Spec (Especificação)

Escreve uma História do Usuário clara, define de 3 a 5 Critérios de Aceite testáveis e as Regras de Negócio (limites, permissões, validações).

3️⃣

Fase Consolidação

Preenche a seção "Especificações Ativas" e atualiza o Backlog marcando a feature como "Pronto para Dev". Avisa que a fase de Engenharia pode começar.

ONION TERMINAL
$ @product ativado Lendo business-context-lite.md... → Entendendo a ideia... Historia: Como usuário logado, quero exportar relatórios em PDF para compartilhar com meu time. → Critérios de Aceite: ✓ PDF com logo da empresa ✓ Máximo 50 linhas por página ✓ Download iniciado em < 3s Gravando spec em business-context-lite.md... ✓ F-01 → "Pronto para Dev"
🛠️ @engineer

Ciclo de Engenharia — O Plano Arquitetural

🔍 Start
📋 Plan
⚙️ Work
✅ Finish

Fase de Início (Start)

Análise de Contexto
  • Lê business-context-lite.md para encontrar a feature marcada como "Pronto para Dev".
  • Lê technical-context-lite.md para entender a stack e arquitetura existentes.
  • Analisa impacto em dependências, segurança e banco de dados antes de planejar qualquer coisa.

Fase de Planejamento (Plan)

⛔ Nunca gere código nesta fase!
  • Escreve a seção "Plano para [Feature]" no technical-context-lite.md.
  • Define a lista de arquivos a serem criados ou modificados.
  • Cria um checklist lógico passo a passo da implementação.
  • Pede aprovação explícita: "Você aprova este plano técnico?"

Fase de Execução (Work)

Após aprovação do plano
  • Cenário B: grava o código diretamente nos arquivos do projeto.
  • Cenário A: entrega blocos de código arquivo por arquivo.
  • Para mudanças longas, divide em partes e reporta o progresso no checklist.
  • Se encontrar erro não previsto, pausa e atualiza o plano antes de continuar.

Fase de Conclusão (Finish)

Entrega & Documentação
  • Fornece instruções de como testar as alterações recém-criadas.
  • Cenário B: atualiza o status da feature para "Feito" direto no business-context-lite.md.
  • Cenário A: instrui o usuário a marcar a feature como "Feito" no arquivo local.
🧠 Personas de Apoio

@meta & @docs — Pesquisa e Sincronismo

🧠

@meta — Knowledge Base

Ativada quando o projeto precisa incorporar uma nova tecnologia, API ou conceito não familiar. Cria documentação técnica mastigada para consumo humano e da própria IA.

  • Pergunta se o usuário tem links/docs ou se usa conhecimento interno.
  • Identifica conceitos, limitações e melhores práticas do tema.
  • Salva o arquivo docs/knowledge-base/[tema].md com estrutura: Visão Geral, Conceitos, Exemplos e Armadilhas (Gotchas).
🔄

@docs — Sync & Engenharia Reversa

Ativada quando há código legado sem documentação Onion. Faz a engenharia reversa do repositório para reconstruir os arquivos de contexto do zero.

  • Cenário B: usa list_dir e view_file para varrer o projeto automaticamente.
  • Identifica a stack real nos arquivos de configuração existentes.
  • Preenche technical-context-lite.md e business-context-lite.md retroativamente com o que o código realmente faz.
📓 @docs — /session

Ciclo de Sessão — Memória Persistente do Projeto

Cada sessão de trabalho é documentada automaticamente. A IA nunca perde o fio da meada.

📁

Estrutura de Sessão

Cada sessão gera uma pasta datada em docs/sessions/YYYY-MM-DD_HHMM_nome/ com 6 artefatos padronizados.

/session startCria a pasta da sessão, preenche context.md com o estado atual do projeto e registra o objetivo.
/session savePersiste decisões, mudanças e notas do que foi feito na sessão em andamento.
/session endFecha a sessão: atualiza o README, gera files-changed.txt e commands-executed.txt, e indexa no README global.

Artefatos gerados por sessão

📄 README.md
Objetivo, resultados, tempo investido
📄 context.md
Estado inicial, motivação, restrições
📄 decisions.md
Decisões arquiteturais com justificativas
📄 changes.md
Arquivos criados e modificados
📄 files-changed.txt
Lista de caminhos para diff rápido
📄 commands-executed.txt
Histórico de comandos executados
💡 Por que importa: Ao retomar o projeto dias depois, a IA lê a última sessão e resgata exatamente o contexto — sem precisar reexplicar nada.
🛡️ A Regra

O Guardião do Fluxo — Impedindo Atalhos

⛔

Invariante Faseada

Nunca gerar código funcional sem antes documentar o produto em business-context-lite.md e planejar a engenharia em technical-context-lite.md. Esta é a lei do Onion.

⚠️ Bloqueio de Bypass

$ Usuário: "Gere o código do relatório PDF agora!" ⚠ ALERTA DE BYPASS DETECTADO Onion → "Detectei tentativa de escrita direta sem especificação. Para manter a integridade, recomendo documentarmos a feature em @product e @engineer primeiro." Prosseguir de forma disciplinada ou forçar?

🔍 Comandos de Diagnóstico

/status ou /healthVarre a pasta docs/, verifica alinhamento de features em progresso com planos técnicos e retorna: OK, Desalinhado ou Incompleto + ações corretivas.
@onion ou "Onion, "Ativa o Orquestrador para assumir o comando, analisar o estado atual e propor o próximo passo lógico do desenvolvimento.
🌐 Ambientes

Cenário A vs. Cenário B

O Onion detecta automaticamente suas ferramentas e adapta o comportamento.

💻

Cenário A — Web Chats

A IA atua como motor lógico sem acesso ao sistema de arquivos local. Entrega adaptativa por plataforma:

  • ChatGPT (Code Interpreter): Executa script Python, cria pasta docs/ e gera link de download do arquivo onion-docs.zip.
  • Claude (Artifacts): Renderiza blocos de artefato individuais para download ou cópia direta.
  • Gemini/outros: Gera markdown completo com caminho do arquivo no cabeçalho do bloco.
⚙️

Cenário B — IDEs Agênticas

A IA possui permissão de leitura e escrita. Automatiza 100% da criação técnica:

  • Antigravity IDE: Edita contextos e código diretamente no disco com aprovação do usuário.
  • Cursor: Usa .cursorrules como âncora do prompt e opera no filesystem local.
  • Claude Code: Agente CLI que lê e escreve no repositório com controle de versão.
  • Zed AI: Integrado ao editor com escrita multi-arquivo coordenada.
🎯 Demo Prática

Demonstração Hands-On do Fluxo

Roteiro para aplicar o Onion Mini ao vivo:

1

Setup Inicial

Fazer o upload de ONION-MASTER-PROMPT.md + business-context-lite.md + technical-context-lite.md + onion-cycles.md no ChatGPT ou Claude.

2

Lançar a Ideia Bruta

Digitar: "Onion, quero criar um botão de exportar para Excel na tela de relatórios."

3

@product em Ação

A IA assume a persona @product, faz perguntas de refinamento e documenta a feature no business-context-lite.md com critérios de aceite.

4

Aprovação de Produto → @engineer

Após aprovar a spec, a IA muda para @engineer e cria o plano técnico detalhado no technical-context-lite.md pedindo aprovação.

5

Aprovação Técnica → Código

Após aprovar o plano, a IA gera o código de forma incremental, relatando cada etapa concluída. No ChatGPT: entrega um ZIP completo para download.
